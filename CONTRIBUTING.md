# Contributing to Roadwatch PH

Thank you for helping improve Roadwatch PH.

## How to Report Issues Effectively

Please use the appropriate **GitHub Issue Form**:
- **Bug Report** for reproducible defects
- **Feature Request** for enhancements
- **Documentation** for content updates
- **Question / Discussion** for general concerns

When filing an issue, include:
- A clear description
- Reproduction steps (for bugs)
- Expected vs actual behavior
- Relevant attachments (screenshots/videos/logs)
- Environment details (OS, browser/app version)
- Location details (route, road name, barangay/city, and GPS if available)
- Date/time observed

Before submitting:
- Search existing issues
- Confirm it is not a duplicate
- Provide enough context for maintainers to act

## Pull Request Process

1. Create a branch from the latest default branch.
2. Make focused, well-scoped changes.
3. Reference related issue(s) in your PR description.
4. Include clear test steps and expected results.
5. Keep commit messages meaningful.
6. Ensure documentation is updated when behavior changes.

## Code of Conduct

Be respectful and constructive in all project interactions:
- Use inclusive and professional language.
- Focus on technical issues, not individuals.
- Assume good intent and collaborate in good faith.

If behavior violates these standards, please contact project maintainers.

## Community Standards

- Prioritize road safety impact and user clarity.
- Favor reproducible reports with specific location/time details.
- Keep discussions helpful and solution-oriented.
- Respect local context in the Philippines when reporting road issues.
